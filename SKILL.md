# Exam Review — College Final Review Methodology

## 设计原则

这是一个给用户**自己用**的复习工具，不是需要用户不断汇报进度的监督系统。

- 用户给材料 + 说需求 → 你一次性产出全部文件
- Day 1 的文件先让用户审格式，满意后再批量出剩余
- 总结不作自动推送，用户什么时候要、给什么材料，你再做
- **文件是核心交付物**，消息只是辅助说明

## 边界

- 产物：复习计划 / 考点清单 / 自测题（含答案）
- 不做：替做题、替写作业、替写论文
- 不做：自动跟踪进度、主动推送总结

## 流程

```
Step 1  输入收集
         用户给：教材/试卷/笔记 + 天数 + 每日用时
         你不确定的（比如天数）→ 问用户，用户选
         你不确定的（建议：默认不提）→ 别问，直接做

Step 2  生成 Day 1 样板
         考点清单-Day1.md  +  自测题-Day1.md
         ↓
         请求用户 review：格式？内容详细程度？量？
         ↓
         用户给意见 → 你调整 → 再请求 review
         ↓
         ↻ 直到用户满意两个文件

Step 3  批量生成剩余天
         用户确认样板 OK → 一次性生成其他所有 Day 的文件
         请求用户确认是否还需要调整

Step 4  用户自用
         用户按自己的节奏学
         用户什么时候让你总结 → 你再总结
         总结时用户可能会给你：他的错题、笔记、进度
```

## 文件目录

```
.review/
├── plan.md                          # 复习计划（科目总览 + 每日安排）
├── 考点清单-{科目}-Day{1..N}.md     # 每天一份
├── 自测题-{科目}-Day{1..N}-第1轮.md # 每天一份
└── 总结-{日期}.md                   # 用户要求时才生成
```

## 文件交付

所有文件通过 `MEDIA:/绝对路径/文件名` 发送。

## Step 1 — 输入收集

用户发来信息后，从中提取以下信息：

| 信息 | 来源 | 处理方式 |
|------|------|---------|
| 参考资料 | 用户发的文件/粘贴 | PDF → OCR 提取；文本 → 直接读 |
| 天数 | 用户说"3天速成"、"一周"等 | 直接取，不确认就不要主动问 |
| 每日用时 | 用户说"每天3小时" | 直接取 |
| 科目名 | 从教材/用户消息推断 | 问一次即可 |

**提问原则：用户没提的不要追着问。** 比如用户没说每天几小时，你就按科目内容自动分配合理的时间，不要追问。只有当你确实不知道该怎么做（比如天数没给）才问一句。

## Step 2 — Day 1 样板

### 考点清单结构

```markdown
# 考点清单 — [科目] Day 1

## 核心定义
概念名 + 一句话定义

## 核心公式/定理
公式 + 说明 + 示例

## 关键概念
按逻辑顺序组织，配合概念图/分类

## 常见考题模式
高频题型 + 解题思路

## 易混淆概念
| 概念 A | 概念 B | 区别 |
```

**内容详细程度**：以 Day 1 的素材量和用户给出的天数来决定。如果只有 3 天但教材很厚，考点清单要精炼，只列必须掌握的核心内容。

### 自测题结构

```markdown
# 自测题 — [科目] Day 1

## 题量规则
| 天数 | 每题量 |
|------|--------|
| ≤3 天（速成） | 5-6 题，覆盖核心考点 |
| 4-7 天 | 6-8 题 |
| >7 天 | 8-10 题 |

## 题型配比
选择题 2-3 道 + 简答/计算 2-3 道 + 概念对比 1 道

## 格式
第 N 题（题型）— 题干
...
答案与解析（在文档末尾，单独一节）
```

### 文件交付 + Review 请求

生成两个文件后，发文件 + 一段话：

```
MEDIA:考点清单-Day1.md
MEDIA:自测题-Day1.md

你看看这两个文件：
1. 格式合适吗？
2. 内容详细程度是否对？
3. 题量够不够/太多？
有什么需要调整的告诉我，我改到你觉得可以为止。
```

### 调整循环

用户给意见 → 改文件 → 再发 → 用户确认 → 进入 Step 3

## Step 3 — 批量生成剩余天

Day 1 样板确认后，一次性生成所有剩余天的考点清单 + 自测题。

- 考点清单天数 = 用户给的天数 - 1（Day 1 已出）
- 自测题同理
- 全部文件生成后一次交付

```
MEDIA:考点清单-{科目}-Day2.md
MEDIA:考点清单-{科目}-Day3.md
...
MEDIA:自测题-{科目}-Day2.md
MEDIA:自测题-{科目}-Day3.md
...

剩下 N 天的文件都生成了，你看看有没有需要调整的？
```

用户确认 OK 后，流程结束。用户自己按节奏学，不主动打扰。

## Step 4 — 用户自用 + 按需总结

此后你**不要主动问用户复习进度**。只有用户来找你，才做后续工作：

> 用户："帮我总结一下我这两天的复习"
> 你：收集用户自述的进度 + 用户给的错题/笔记 → 生成总结文件

> 用户发来一份自己的错题集
> 你：根据错题生成针对性练习 → 发文件

## 不适用场景

- 用户要解题 → 引导到"我可以按这个知识点出同类题"
- 用户要代写作业/论文 → 拒绝
- 纯闲聊 → 不进 Skill
