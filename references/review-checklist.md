# Exam Review 质检清单

## Plan 自查段（主 Agent 内联，不开 SubAgent）

Plan 写完后主 Agent 就地逐项检查：

1. **科目和考试日期是否完整？** 有没有遗漏用户提到的科目？
2. **掌握度是否已确认？** 没有掌握度的计划是拍脑袋的，每科必须确认当前水平。
3. **每日学习时长是否合理？** 超过 8 小时要自动降至 8 小时并提示。
4. **优先级排序是否合理？** 紧迫度 = (考试天数 - 今天) / 掌握度，低掌握+临近考试 = 最高优先级。
5. **这份计划用户能看懂吗？** 避免模糊表述，每条任务可执行。

按结论修复 plan.md，然后进入 Checkpoint 1。

---

## Test 自查段（主 Agent 内联，不开 SubAgent）

自测出题后，主 Agent 逐项检查：

1. **题目质量**：答案是正确的吗？选项中的干扰项合理吗？
2. **评级是否记录？** 每道题答完后必须记录掌握度评级。
3. **正确率是否计算？** 本次自测的总正确率、各知识点正确率。
4. **薄弱环节是否标记？** 正确率 < 60% 的知识点标记为薄弱。
5. **下一步行动明确吗？** 自测结束后用户知道该做什么。

按结论修复 topics.json / progress.json，然后进入 Checkpoint 2。

## 间隔重复计算规则

| 评级 | 掌握度 | 下次回顾间隔 |
|------|-------|------------|
| Mastered | 90-100% | 7 天后 |
| Got it | 70-89% | 3 天后 |
| Needs work | 40-69% | 1 天后 |
| No idea | 0-39% | 现在复习 → 1 天后 |

计算方式：
- 首次未掌握的知识点，记入「今日补学」队列
- 自测正确率 = 答对题数 / 总题数
- 薄弱环节列表 = 掌握度 < 60% 的所有知识点
