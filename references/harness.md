# Harness 视角

本 Skill 的重点不是"提示词"，而是一个小型 harness。它要回答六个问题：

| Harness 部分 | 本 Skill 解决的问题 | 设计手段 |
|---|---|---|
| 上下文管理 | 模型到底看到了什么 | 渐进加载：Intake 只看模式识别，Plan 才加载科目列表，Test 才读 topic 数据 |
| 工具系统 | 能处理什么输入/输出 | 用户粘贴笔记 / 文件读取 / PDF OCR；输出复习计划、考点清单、自测题 |
| 执行编排 | 下一步该做什么 | 分 Phase、检查点、先做计划再学习→自测→回顾的复习闭环 |
| 状态与记忆 | 决策如何跨步骤保持 | `.review/` 工作区：`plan.md`（复习计划）、`topics.json`（知识点掌握度）、`progress.json`（复习进度） |
| 评估与观测 | 怎么知道复习效果好不好 | 自测正确率计算、薄弱环节标记、间隔重复间隔 |
| 约束与恢复 | 跑偏后怎么修 | Checkpoint 用户确认制、不做解题器（引导回复习模式）、计划太激进自动调整 |

## 工作区文件（.review/）

Agent **不应依赖聊天上下文记住关键决策**。跨阶段决策落盘到下列文件：

```text
.review/
  plan.md              # 复习计划：科目、考试日期、每日时间表、优先级
  topics.json          # 知识点库：每个知识点名称、评级、掌握度(0-100)、错题记录、下次回顾时间
  progress.json        # 复习进度：已完成轮次、正确率趋势、薄弱环节列表
```

长会话中如果不确定某个已确认的决策，**回读这些文件**，不要凭记忆重新发明。

## 一句话定位

> Exam Review 帮大学生用科学方法论管理复习：先看清楚有哪些考试再定计划，学完就测、测完就记、记完再回顾——不是替你做題，是帮你把复习节奏管起来。
